# Hermes Multi-Agent Orchestrator

**Мульти-агентная оркестрация для Hermes Agent на бесплатных моделях OpenCode Zen.**

Замени одного агента — командой агентов. Преврати последовательность — в параллель. Часы — в минуты. 5 специализированных ролей, 3 режима оркестрации.

## Модели (OpenCode Zen — Бесплатный тир)

| Роль | Модель | Сильные стороны |
|------|--------|----------------|
| 🎯 **Архитектор** | Big Pickle Med | Архитектура, креатив, структура |
| 📋 **Планировщик / 🔍 Исследователь** | Deepseek V4 Flash Free Max | Скорость, сбалансированность, надёжность |
| 💻 **Кодер** | North Mini Code Free Med | Код, технические задачи |
| ✍️ **Синтезатор** | Mimo V2.5 Free Med | Генерация, креатив, сборка |
| 🔎 **Ревьюер** | Nemotron 3 Ultra Free Max | Глубокий анализ (⚠️ может таймаутить) |

## Режимы

| Режим | Паттерн | Когда использовать |
|-------|---------|-------------------|
| 🤖 **Council** | Все агенты параллельно → Арбитр синтезирует | Исследование, сравнение |
| 🔧 **Pipeline** | Последовательно A → B → C → D | Code review, документы |
| 🎭 **Hybrid** | Сначала параллель, потом последовательно | Сложные смешанные задачи |

## Быстрый старт

Это **скилл для Hermes Agent**. Работает через `delegate_task`.

### Пример: Параллельное исследование (Council)

> Пользователь: *«Сравни LangChain, CrewAI и AutoGen»*

1. 🎯 Архитектор (Big Pickle) — декомпозиция
2. 📋 Планировщик (Deepseek) — детальный план
3. 🔍 3 Исследователя (Deepseek) — параллельный поиск
4. ✍️ Синтезатор (Mimo) — сборка сравнения
5. 🔎 Ревьюер (Nemotron) — финальная проверка

### Пример: Code Pipeline

> Пользователь: *«Проверь и исправь этот код»*

1. 🎯 Архитектор (Big Pickle) — анализ структуры
2. 📋 Планировщик (Deepseek) — план изменений
3. 💻 Кодер (North Mini Code) — исправления
4. 🔎 Ревьюер (Nemotron) — финальное ревью

## Питфоллы

- **Nemotron 3 Ultra** таймаутит на длинных контекстах → замена на Deepseek
- **Параллельные агенты изолированы** — зависимые задачи вторым раундом
- **Объявлять активацию ДО работы** — иначе пользователь видит молчание
- **Не выводить сырые результаты агентов** — всегда синтезировать
